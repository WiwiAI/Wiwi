# Security Policy

## Поддерживаемые версии

Сейчас активная ветка: `main`.

## Сообщение об уязвимостях

Пожалуйста, **не публикуй секреты и детали эксплуатации в публичных issues**.

Как сообщать:
1. Открой private security advisory в GitHub (если доступно),
2. или создай issue с минимальными деталями и пометкой `security` без PoC/секретов,
3. после чего мы переведём общение в приватный канал.

## Что считается критичным

- утечка токенов/ключей;
- обход авторизации;
- удалённое выполнение кода;
- небезопасная обработка пользовательского ввода;
- уязвимости в supply chain (зависимости/скрипты).

## Рекомендации контрибьюторам

- никогда не коммить `.env`, токены, приватные ключи;
- используй плейсхолдеры в примерах (`YOUR_TOKEN_HERE`);
- при утечке — немедленно ротируй токены.
